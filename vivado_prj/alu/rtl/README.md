# `top.v` ALU 说明

## 1. 模块作用

`top.v` 实现了一个 **32 位纯组合逻辑 ALU**。

- 输入两个 32 位操作数 `a`、`b`
- 输入一个 5 位控制信号 `alu_op`
- 根据 `alu_op` 选择不同的运算
- 输出 32 位结果 `y`
- 同时输出若干状态标志位

这个模块里 **没有时钟、没有复位、没有寄存器时序逻辑**，所以是一个组合模块。

---

## 2. 端口说明

### 输入端口

- `a[31:0]`
  - 第一个 32 位输入操作数

- `b[31:0]`
  - 第二个 32 位输入操作数
  - 在移位类操作中，`b[4:0]` 还会被当作移位位数使用

- `alu_op[4:0]`
  - ALU 的功能选择信号
  - 不同编码对应不同运算

### 输出端口

- `y[31:0]`
  - ALU 运算结果

- `zero`
  - 当 `y == 0` 时为 `1`

- `negative`
  - 结果最高位 `y[31]`
  - 在有符号语义下可理解为“结果为负”

- `carry_out`
  - 加法/减法等运算的进位或借位标志

- `overflow`
  - 有符号运算溢出标志

- `parity_even`
  - 结果 `y` 的偶校验
  - 当 `y` 中 `1` 的个数为偶数时，该位为 `1`

- `eq`
  - 输入 `a` 和 `b` 是否相等

- `lt_signed`
  - 有符号比较下，`a < b` 时为 `1`

- `lt_unsigned`
  - 无符号比较下，`a < b` 时为 `1`

---

## 3. 内部辅助信号说明

- `a_signed`
  - 把 `a` 解释为有符号数

- `b_signed`
  - 把 `b` 解释为有符号数

- `shamt`
  - 移位位数，等于 `b[4:0]`

- `mul_u`
  - `a * b` 的无符号 64 位乘法结果

- `mul_s`
  - `a * b` 的有符号 64 位乘法结果

- `sum_ext`
  - 扩展到 33 位的加法结果，用来保留进位

- `diff_ext`
  - 扩展到 33 位的减法结果，用来判断借位

- `carry_r`
  - 内部保存 `carry_out`

- `overflow_r`
  - 内部保存 `overflow`

---

## 4. 功能码说明

`alu_op` 的定义如下：

| `alu_op` | 名称 | 功能说明 |
|---|---|---|
| `5'd0`  | `ALU_ADD` | 加法，`y = a + b` |
| `5'd1`  | `ALU_SUB` | 减法，`y = a - b` |
| `5'd2`  | `ALU_AND` | 按位与 |
| `5'd3`  | `ALU_OR` | 按位或 |
| `5'd4`  | `ALU_XOR` | 按位异或 |
| `5'd5`  | `ALU_NOR` | 按位或后取反 |
| `5'd6`  | `ALU_SLL` | 逻辑左移，`a << b[4:0]` |
| `5'd7`  | `ALU_SRL` | 逻辑右移，`a >> b[4:0]` |
| `5'd8`  | `ALU_SRA` | 算术右移 |
| `5'd9`  | `ALU_ROL` | 循环左移 |
| `5'd10` | `ALU_ROR` | 循环右移 |
| `5'd11` | `ALU_SLT` | 有符号比较，小于则输出 `1` |
| `5'd12` | `ALU_SLTU` | 无符号比较，小于则输出 `1` |
| `5'd13` | `ALU_MAX` | 输出有符号较大值 |
| `5'd14` | `ALU_MIN` | 输出有符号较小值 |
| `5'd15` | `ALU_ABS` | 取 `a` 的绝对值 |
| `5'd16` | `ALU_BYTE_SWAP` | 字节翻转 |
| `5'd17` | `ALU_POPCOUNT` | 统计 `a` 中 `1` 的个数 |
| `5'd18` | `ALU_CLZ` | 统计前导零个数 |
| `5'd19` | `ALU_CTZ` | 统计尾随零个数 |
| `5'd20` | `ALU_MUL_LOW` | 无符号乘法低 32 位 |
| `5'd21` | `ALU_MUL_HIGH` | 有符号乘法高 32 位 |
| `5'd22` | `ALU_AVG` | 平均值 `(a & b) + ((a ^ b) >> 1)` |
| `5'd23` | `ALU_PACK` | 拼接 `{a[15:0], b[15:0]}` |
| `5'd24` | `ALU_SAT_ADD` | 饱和加法 |
| `5'd25` | `ALU_SAT_SUB` | 饱和减法 |
| `5'd26` | `ALU_BITREV` | 位翻转 |
| `5'd27` | `ALU_EQ` | 若 `a == b` 输出 `1`，否则输出 `0` |
| `5'd28` | `ALU_XNOR` | 按位同或 |
| `5'd29` | `ALU_PASS_A` | 直接输出 `a` |
| `5'd30` | `ALU_PASS_B` | 直接输出 `b` |
| 其他 | `default` | 默认输出 `0` |

## 5. 主要功能逐项解释

### 5.1 加减法

- `ALU_ADD`
  - 输出 `a + b`
  - `carry_out` 表示最高位进位
  - `overflow` 表示有符号加法溢出

- `ALU_SUB`
  - 输出 `a - b`
  - `carry_out` 在这里表示“无借位”
  - `overflow` 表示有符号减法溢出

---

### 5.2 逻辑运算

- `ALU_AND`
  - 按位与

- `ALU_OR`
  - 按位或

- `ALU_XOR`
  - 按位异或

- `ALU_NOR`
  - 先按位或，再整体取反

- `ALU_XNOR`
  - 按位异或后取反

---

### 5.3 移位与循环移位

- `ALU_SLL`
  - 逻辑左移，低位补 0

- `ALU_SRL`
  - 逻辑右移，高位补 0

- `ALU_SRA`
  - 算术右移，高位补符号位

- `ALU_ROL`
  - 循环左移，移出去的高位回到低位

- `ALU_ROR`
  - 循环右移，移出去的低位回到高位

---

### 5.4 比较类

- `ALU_SLT`
  - 有符号比较
  - 若 `a < b`，则 `y = 32'h0000_0001`
  - 否则 `y = 0`

- `ALU_SLTU`
  - 无符号比较

- `ALU_EQ`
  - 若 `a == b`，则 `y = 1`

- `eq`
  - 这是额外输出标志位
  - 不依赖 `alu_op`
  - 只要 `a == b` 就一直为 `1`

---

### 5.5 选择类与数值处理

- `ALU_MAX`
  - 输出 `a` 和 `b` 中较大的有符号数

- `ALU_MIN`
  - 输出 `a` 和 `b` 中较小的有符号数

- `ALU_ABS`
  - 输出 `a` 的绝对值
  - 若 `a = 32'h8000_0000`，理论绝对值超出 32 位有符号范围
  - 此时会置 `overflow = 1`

- `ALU_PASS_A`
  - 直接透传 `a`

- `ALU_PASS_B`
  - 直接透传 `b`

---

### 5.6 数据重排类

- `ALU_BYTE_SWAP`
  - 交换字节顺序
  - 例如：
  - 输入 `a = 32'h1122_3344`
  - 输出 `y = 32'h4433_2211`

- `ALU_PACK`
  - 输出 `{a[15:0], b[15:0]}`
  - 结果高 16 位来自 `a` 的低 16 位
  - 结果低 16 位来自 `b` 的低 16 位

- `ALU_BITREV`
  - 把 32 位比特顺序完全翻转
  - `a[31]` 变成 `y[0]`
  - `a[0]` 变成 `y[31]`

---

### 5.7 位统计类

- `ALU_POPCOUNT`
  - 统计 `a` 中 bit 为 `1` 的数量
  - 结果范围是 `0` 到 `32`

- `ALU_CLZ`
  - 统计前导零个数
  - 例如 `a = 32'h0000_00F0`，前导零较多
  - 若 `a = 0`，结果定义为 `32`

- `ALU_CTZ`
  - 统计尾随零个数
  - 若 `a = 0`，结果定义为 `32`

---

### 5.8 乘法与平均值

- `ALU_MUL_LOW`
  - 计算无符号乘法 `a * b`
  - 取结果低 32 位

- `ALU_MUL_HIGH`
  - 计算有符号乘法 `a * b`
  - 取结果高 32 位

- `ALU_AVG`
  - 计算平均值
  - 使用位运算公式避免直接 `(a + b) >> 1` 的潜在溢出问题

---

### 5.9 饱和运算

- `ALU_SAT_ADD`
  - 饱和加法
  - 如果有符号加法溢出：
  - 正溢出时输出 `32'h7FFF_FFFF`
  - 负溢出时输出 `32'h8000_0000`

- `ALU_SAT_SUB`
  - 饱和减法
  - 如果有符号减法溢出：
  - 正溢出时输出 `32'h7FFF_FFFF`
  - 负溢出时输出 `32'h8000_0000`

---

## 6. 标志位说明

### `zero`

- `y == 0` 时为 `1`

### `negative`

- 等于 `y[31]`
- 主要用于有符号结果判断

### `carry_out`

- 对加法来说表示进位
- 对减法来说表示无借位
- 对其他大多数操作默认是 `0`

### `overflow`

- 表示有符号溢出
- 主要在以下操作中有意义：
- `ALU_ADD`
- `ALU_SUB`
- `ALU_ABS`
- `ALU_SAT_ADD`
- `ALU_SAT_SUB`

### `parity_even`

- 对结果 `y` 做偶校验

### `eq`

- `a == b` 时为 `1`

### `lt_signed`

- 按有符号数比较 `a < b`

### `lt_unsigned`

- 按无符号数比较 `a < b`

---

## 7. 使用举例

### 例 1：加法

- `a = 32'd10`
- `b = 32'd20`
- `alu_op = 5'd0`
- 结果 `y = 32'd30`

### 例 2：有符号比较

- `a = -5`
- `b = 3`
- `alu_op = 5'd11`
- 因为 `-5 < 3`
- 所以 `y = 32'd1`

### 例 3：字节翻转

- `a = 32'h1234_5678`
- `alu_op = 5'd16`
- `y = 32'h7856_3412`

### 例 4：位计数

- `a = 32'h0000_00FF`
- `alu_op = 5'd17`
- `y = 32'd8`

---

## 8. 设计特点总结

这份 ALU 的特点是：

- 纯组合逻辑，结构清晰
- 功能比基础 ALU 更丰富
- 既支持普通算术逻辑运算，也支持统计和重排类运算
- 带有常见状态标志位，方便后续扩展到 CPU 或更大模块

如果后面你要继续做实验，比较适合继续补：

- `testbench`
- `Vivado` 工程约束
- 功能仿真激励
- 操作码定义头文件
